fix(edge): simplificar edge-fix para evitar romper estilos CSS

Remover logica agresiva de recarga de CSS que causaba que el header
perdiera estilos en Edge despues de navegacion.

Problema:
- edge-fix.js estaba recargando CSS de ABP/Blazorise
- Causaba que header perdiera estilos (volvía a default ABP)
- Fix era demasiado agresivo monitoreando navegacion
- Logs mostraban: "Recargando stylesheet despues de navegacion"

Causa:
- MutationObserver detectaba carga inicial como navegacion
- Forzaba recarga de CSS que ya estaban correctos
- Race condition entre ABP styles y recarga manual

Solucion:
- SOLO suprimir warnings (filtrado de console.warn)
- NO tocar CSS en absoluto
- NO monitorear navegacion
- NO forzar recargas

edge-fix.js ahora:
- Detecta Edge
- Suprime warnings de preload
- Suprime warnings de IdentityClientConfiguration
- FIN - nada mas

Beneficios:
- Header mantiene estilos correctamente
- Sin race conditions
- Sin overhead de monitoreo
- Fix minimo y seguro

Testing:
- Verificar en Edge que header mantenga estilos
- Verificar que warnings de preload no aparezcan
- Navegar multiples veces sin problemas
